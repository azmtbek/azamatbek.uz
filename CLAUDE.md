# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Stack

- **Framework**: Next.js 16 (App Router) ‚Äî server components by default
- **Runtime**: React 19, TypeScript
- **Styling**: Tailwind CSS v4 ‚Äî configured via `@tailwindcss/postcss` plugin, no `tailwind.config.*` file
- **Components**: shadcn/ui (config in `components.json`)
- **Linting/Formatting**: Biome (config in `biome.json`) ‚Äî not ESLint, not Prettier
- **Testing**: Vitest
- **Package manager**: pnpm only ‚Äî `npm` and `yarn` are blocked by `preinstall` hook

## Commands

```bash
pnpm dev          # Dev server at http://localhost:3000
pnpm build        # Production build (runs generate-feeds.ts first via prebuild)
pnpm start        # Serve production build
pnpm test         # Run Vitest (server tests ‚Äî needs dev server running separately)
pnpm test:build   # Build then run build smoke tests
pnpm lint         # Biome lint
pnpm format       # Biome format
pnpm check        # Biome lint + format
```

## Architecture

### Routing

Routes live in `src/app/` (Next.js App Router convention).

- `src/app/layout.tsx` ‚Äî root layout: HTML shell, `<Header>`, RSS link in `<head>`
- `src/app/page.tsx` ‚Äî home page (`/`)
- `src/app/blog/page.tsx` ‚Äî blog listing (`/blog`)
- `src/app/blog/[slug]/page.tsx` ‚Äî single post (`/blog/:slug`)
- `src/app/blog/tags/[tag]/page.tsx` ‚Äî tag filter (`/blog/tags/:tag`)
- `src/app/projects/page.tsx` ‚Äî projects (`/projects`)
- `src/app/contact/page.tsx` ‚Äî contact (`/contact`)
- `src/app/not-found.tsx` ‚Äî 404 page

### Adding a route

```tsx
// src/app/about/page.tsx
import type { Metadata } from "next"

export const metadata: Metadata = { title: "About ‚Äî Azamatbek" }

export default function AboutPage() {
  return <main>About</main>
}
```

### Page metadata

Use `generateMetadata()` for dynamic metadata (e.g., blog posts), or export a static `metadata` object for fixed pages:

```tsx
export async function generateMetadata({ params }: Props): Promise<Metadata> {
  const post = await getPostBySlug((await params).slug)
  return { title: `${post?.title} ‚Äî Azamatbek` }
}
```

### Static params for dynamic routes

Use `generateStaticParams()` to pre-render dynamic routes at build time:

```tsx
export function generateStaticParams() {
  return getAllPosts().map((p) => ({ slug: p.slug }))
}
```

### Blog content

- Posts live in `public/posts/*.md` with frontmatter (`title`, `date`, `description`, `tags`)
- `src/lib/posts.ts` exports `getAllPosts()` and `getPostBySlug(slug)`
- `postsDir` is `path.join(process.cwd(), "public/posts")` ‚Äî works in both dev and production
- RSS and sitemap are generated into `public/` by `scripts/generate-feeds.ts`, which runs automatically via `prebuild`

### Styles

Global CSS is at `src/app/globals.css`. Tailwind v4 uses CSS-based config (`@theme` block inside that file) ‚Äî there is no JS config file.

### shadcn/ui

Add components with:
```bash
pnpm dlx shadcn@latest add <component>
```

Components land in `src/components/ui/`.

## Key gotchas

- `next-env.d.ts` ‚Äî auto-generated by Next.js, do not touch
- `src/components/Header.tsx` is `"use client"` because it uses `usePathname()` for active link detection
- Tailwind v4: no `tailwind.config.ts`, utilities defined in CSS via `@theme`; PostCSS plugin is `@tailwindcss/postcss`
- Biome uses tabs for indentation and double quotes for JS strings (see `biome.json`)
- `public/posts/` is where blog posts are stored ‚Äî the `public/` directory is served as static assets by Next.js
- `/blog/[slug]` returns 200 with "Post not found" content for unknown slugs (intentional ‚Äî not a hard 404)
